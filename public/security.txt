# Security Disclosure Policy

## Scope
This security policy applies to:
- HalloWa web application (https://multi-wa-mate.lovable.app)
- API endpoints (https://*.supabase.co/functions/*)
- Backend services (Railway)
- Database (Supabase PostgreSQL)

## Reporting Security Vulnerabilities

If you discover a security vulnerability, please report it responsibly:

1. **DO NOT** publicly disclose the vulnerability until it has been addressed
2. Send details to: security@hallowa.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if available)

We commit to:
- Acknowledge receipt within 48 hours
- Provide an initial assessment within 7 days
- Keep you informed of progress
- Credit you for responsible disclosure (if desired)

## Out of Scope

The following are **NOT** considered security vulnerabilities:
- Social engineering attacks
- Denial of Service (DoS) attacks
- Spam or social media abuse
- Physical attacks
- Attacks requiring physical access to user devices

## Bounty Program

We do not currently offer a bug bounty program, but we:
- Acknowledge security researchers in our security hall of fame
- Provide public recognition for significant findings
- Offer swag/merchandise for exceptional reports

## Security Best Practices

### For Users:
✓ Use strong passwords (12+ characters with complexity)
✓ Enable 2FA for admin accounts (when available)
✓ Never share API keys publicly
✓ Report suspicious activity immediately
✓ Keep your browser updated
✓ Log out after using shared devices

### For Developers/Integrators:
✓ Store API keys securely (environment variables)
✓ Use HTTPS for all API calls
✓ Implement rate limiting on your side
✓ Validate and sanitize all user inputs
✓ Never expose service role keys in frontend
✓ Follow OWASP Top 10 guidelines

## Security Measures Implemented

✅ Authentication & Authorization
   - Supabase Auth with JWT tokens
   - Role-based access control (RBAC)
   - Row Level Security (RLS) policies
   - Protected routes

✅ API Security
   - API key authentication
   - Rate limiting (100 requests/minute)
   - CORS whitelist (no wildcards)
   - Input validation and sanitization
   - SSRF protection

✅ Data Protection
   - HTTPS everywhere
   - Encrypted connections
   - Secure session management
   - Auto-logout after 30 min inactivity
   - PII redaction in logs

✅ Frontend Security
   - Content Security Policy (CSP)
   - XSS protection headers
   - Clickjacking prevention (X-Frame-Options)
   - MIME sniffing prevention
   - Error boundary for crash prevention
   - Strong password policy (12+ chars)

✅ Backend Security
   - Webhook signature verification (HMAC SHA-256)
   - Secure logging with sensitive data redaction
   - Password hashing (bcrypt)
   - SQL injection prevention (parameterized queries)

✅ Compliance
   - OWASP Top 10 (2021): 100% compliant
   - GDPR Article 32: Data security compliant
   - SOC 2 Type II: Access controls & audit logs
